==================
Primeiro Prompt 
==================

Quero que você atue como um desenvolvedor sênior extremamente didático e pragmático, especializado em Python, FastAPI, HTML simples embutido e integrações com Azure AI Services.
Seu objetivo é me ajudar a criar uma aplicação demo MUITO simples chamada:
"STT + resumo com LLM"

Contexto:
- Esta app será usada em um workshop interno de introdução à IA.
- O objetivo é demonstrar, de forma rápida e visual, um fluxo simples de:
  1. capturar áudio pela interface usando microfone
  2. permitir que o usuário grave sua fala no navegador
  3. enviar esse áudio gravado para o backend
  4. transcrever o áudio com Speech to Text
  5. exibir a transcrição na tela
  6. enviar a transcrição para um LLM no Azure OpenAI
  7. gerar um resumo curto e didático
  8. exibir o resumo na tela

IMPORTANTE:
- O foco é didática e clareza, não produção.
- O projeto precisa ser extremamente simples.
- O código deve ser fácil de entender para pessoas com pouca familiaridade com Python e IA.
- Não usar arquitetura complexa.
- Não usar frontend separado.
- O HTML pode ser embutido no próprio backend, ou usar template mínimo.
- Tudo deve rodar localmente.
- O código deve estar pronto para eu demonstrar num workshop e também gravar a codificação depois.
- Quero um app pequeno, com poucas etapas e poucos arquivos.

==================================================
OBJETIVO FUNCIONAL DA APP
==================================================

A app deve ter uma interface simples com:

- um título
- uma breve descrição
- um botão para iniciar gravação do microfone
- um botão para parar gravação
- um botão para enviar/processar o áudio gravado
- uma área para mostrar:
  - status da gravação
  - status do processamento
  - transcrição
  - resumo

Fluxo esperado:
1. usuário clica para iniciar gravação
2. a aplicação usa o microfone do navegador
3. o usuário fala
4. o usuário para a gravação
5. o áudio gravado é enviado para o backend
6. backend chama o serviço de Speech to Text da Azure
7. backend obtém a transcrição
8. backend chama o Azure OpenAI com um prompt simples de resumo
9. backend devolve para a tela:
   - texto transcrito
   - resumo em tópicos curtos

==================================================

O resumo gerado pelo Azure OpenAI deve ser simples e útil.
Exemplo de comportamento esperado:
- resumir em 3 a 5 tópicos
- linguagem clara
- tom corporativo amigável
- destacar ideia principal
- não inventar informações
- se a transcrição vier vazia ou ruim, avisar isso

Sugestão de instrução:
“Resuma a transcrição em 3 a 5 tópicos curtos, em português do Brasil, com linguagem clara e objetiva. Se a transcrição estiver incompleta ou pouco compreensível, diga isso explicitamente.”

DETALHES IMPORTANTES SOBRE A UI
==================================================

A interface pode ser bem simples, mas quero algo minimamente agradável:

- layout limpo

==================================================
DETALHES IMPORTANTES SOBRE CAPTURA DE ÁUDIO
==================================================

A captura de áudio deve ser feita no navegador, da forma mais simples e didática possível.

Preferências:
- usar JavaScript simples
- usar APIs nativas do navegador, como MediaRecorder, se isso for a opção mais simples
- gravar áudio curto
- permitir enviar o blob/arquivo gravado para o backend
- manter o fluxo fácil de entender

Não quero:
- streaming em tempo real
- websocket
- complexidade de áudio em tempo real
- interface sofisticada

==================================================
DETALHES IMPORTANTES SOBRE SPEECH TO TEXT
==================================================

A implementação do Speech to Text deve ser a forma mais simples e didática possível.
Se houver mais de uma forma de integrar, escolha a mais simples para workshop.

Se você precisar assumir algo, prefira:
- áudio gravado e enviado pelo frontend
- processamento síncrono simples
- caso de uso demonstrativo
- sem pipeline avançado
